# IPv6-only / IPv6-mostly
## NAT64/DNS64
- IPv6-onlyのネットワークから、IPv4ネットワークのサービスに接続できるようにする仕組み

### 動作フロー
1. ホストはDNS64を有効化しているDNSに対し、宛先ホストのIPアドレスを問い合わせる
2. DNSクエリを受信したDNS64サーバは、取得したIPv4アドレスとIPv4-IPv6変換アドレス用のprefixを使用して
   IPv6アドレスを合成し、送信元ホストに返す
3. ホストが`2`のアドレス宛にパケットを送る
    - 通信の際、利用するIPv4-IPv6変換アドレスが登録されたNAT64対応ネットワークを利用する
4. NAT64は設定されたprefixが宛先となっているパケットを取得してそのパケットのIPv6ヘッダをIPv4ヘッダに置き換える
5. 通信に応答があった場合、NAT64はパケットのIPv4ヘッダをIPv6ヘッダに付け替え、
   元のIPv4-IPv6変換アドレスを送信元として通信を開始した端末へ返送する

### PREF64
- 送信元ホスト自身がDNS64の存在を検出し、自らアドレスの合成を行う仕組み
- ホスト自身がネットワークで利用されているIPv4-IPv6変換アドレス用のプリフィクス情報を提供する機能を持つ
- Pref64::/n Discovery (RFC7050)

### 464XLAT
- IPv4専用アプリケーションをIPv6-onlyのネットワーク上で動作させるための変換機構 (RFC 6877)
  - IPv4アプリケーション -> [CLAT: IPv4 -> IPv6] -> IPv6ネットワーク -> [PLAT: IPv6 -> IPv4] -> IPv4サーバ
  - CLAT (Customer-side Translator) : クライアント端末でIPv4 -> IPv6変換を行うステートレスNAT46
  - PLAT (Provider-side Translator) : ネットワーク側 (NAT64ゲートウェイ) でIPv6 -> IPv4変換を行うステートフルNAT64

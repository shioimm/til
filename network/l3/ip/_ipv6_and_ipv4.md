# IPv6 / IPv4 共存技術
## デュアルスタック
- ホストがIPv4とIPv6を両方持ち、接続先に応じて使い分ける
- 実装がシンプルで信頼性が高い、IPv4/IPv6どちらとも直接通信できる、
- IPv4アドレスを消費し続ける、管理するプロトコルスタックが2つになる

## トランスレータ
- IPv6-onlyクライアントからIPv4-onlyサーバへアクセスする際、DNSや中間機器やホスト自身がアドレスとパケットを変換する
  - DNS64: IPv4-onlyサーバのAAAAクエリに対し、DNS64リゾルバがAレコードを元にAAAAレコードを合成して返す仕組み
  - NAT64: IPv4-onlyサーバに対するIPv6パケットを、ネットワーク上のNAT64装置がIPv4パケットに変換する仕組み
  - 464XLAT: IPv4-onlyサーバに対するIPv6パケットを、クライアント自身がIPv4パケットに変換する仕組み
    - CLAT: アプリケーションが送出したIPv4パケットをIPv6パケットに変換し、IPv4アドレスをIPv6アドレスへ合成する
    - PLAT: ネットワーク上のNAT64装置が、CLATから届いたIPv6パケットをIPv4パケットに変換してサーバへ転送する
- IPv6 Onlyネットワークを実現できる
- エンドツーエンドの透過性が失われる、NAT64装置がSPOF・ボトルネックになりうる

### NAT64/DNS64の動作フロー
- NAT64/DNS64を利用するネットワークは、アドレスの変換に利用するIPv4-IPv6変換アドレス用プリフィクスが割り当てられ、
  そのプリフィクス宛の通信がNAT64に到達するように構築される
- 個々の端末はDNS64サーバに対してアドレス解決の問い合わせを行うように設定される

1. IPv6-onlyホストが、IPv4-onlyサーバのAAAAレコードを問い合わせる
2. 権威サーバはリゾルバに空の応答を返す
3. DNS64対応リゾルバは、空のAAAA応答に対し、Aレコードを問い合わせ直す
4. DNS64対応リゾルバは、取得したA応答にNAT64用のプレフィックスを合成して擬似的なIPv6アドレスを生成する
5. ホストが合成済みIPv6アドレス宛にパケットを送る
    - 通信の際、利用するIPv4-IPv6変換アドレスが登録されたNAT64対応ネットワークを利用する
6. NAT64は設定されたprefixが宛先となっているパケットのIPv6ヘッダをIPv4ヘッダに置き換えてサーバへ転送する
7. サーバから応答があった場合、NAT64はパケットのIPv4ヘッダをIPv6ヘッダに付け替え、
   元のIPv4-IPv6変換アドレスを送信元として通信を開始した端末へ返送する

- 合成アドレスを得たもののNATがNAT64に対応していない場合、サーバに到達できず、通信が失敗する

#### PREF64
- NAT64がIPv4アドレスを埋め込むために使用する特別なIPv6プレフィックス (Pref64::/n Discovery [RFC7050])
- well-knownプレフィックスとして64:ff9b::/96がIANAにより定められている
- ISPが独自のプレフィックスを使う場合、ルータ広告オプションでクライアントに通知する ([RFC 8781])

### 464XLAT
- DNSを経由してアドレスを解決しないケースにおいて、
  IPv4専用アプリケーションをIPv6-onlyのネットワーク上で動作させるための変換機構 (RFC 6877)
  - IPv4アドレスを直接利用するアプリケーション
  - IPv4のソケットAPIを直接使うレガシーアプリケーション
- IPv4アプリケーション -> [CLAT: IPv4 -> IPv6] -> IPv6ネットワーク -> [PLAT: IPv6 -> IPv4] -> IPv4サーバ

1. アプリケーションがIPv4パケットを送信
2. CLATがPREF64を使ってアドレス合成し、IPv4パケットをIPv6パケットに変換する
3. CLATがIPv6網を経由してパケットを転送
4. NAT64 (PLAT) が設定されたprefixが宛先となっているパケットのIPv6ヘッダをIPv4ヘッダに置き換えてサーバへ転送する
5. サーバから応答があった場合、NAT64はパケットのIPv4ヘッダをIPv6ヘッダに付け替え、
   元のIPv4-IPv6変換アドレスを送信元として通信を開始した端末へ返送する
5. CLATがIPv6パケットをIPv4パケットに変換し、アプリケーションへ渡す

## トンネリング
- あるプロトコルのパケットを別のプロトコルのパケットにカプセル化して転送する
  - 6in4 / 6rd: IPv6パケットをIPv4パケットにカプセル化して転送する仕組み (減少傾向)
  - DS-Lite: IPv4パケットをIPv6パケットにカプセル化して転送し、出口のCGNATでIPv4アドレスを共有する
  - MAP-E: IPv4パケットをIPv6パケットにカプセル化、数式的なルールでIPv4アドレスとポート番号をユーザに割り当てる
  - MAP-T:MAP-Eのカプセル化をトランスレーションに置き換えたもの
- 既存インフラを大きく変えずにIPv6を導入できる
- カプセル化によるオーバーヘッドがある、方式が多岐にわたり互換性がない

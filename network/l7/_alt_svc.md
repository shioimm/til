# Alt-Svc
- HTTPレスポンスにおいて、次回以降のリクエスト時に、そのオリジンのサーバとして別のネットワーク上
  の場所別の接続先・別のプロトコルでも同じリソースにアクセスできることを示す
- サーバはAlt-SvcヘッダまたはHTTP/2のALTSVCフレームを用いて"h3"ALPNトークンによるHTTP/3エンドポイントを広告できる

```text
// e.g.
// このオリジンは同じホストの443番ポートでHTTP/3でも利用可能・この情報は86400秒キャッシュ可能
Alt-Svc: h3=":443"; ma=86400
```

- 主な用途はHTTP/3が利用可能であることを発見すること
  - HTTP/2ではTLSハンドシェイク内のALPNで"h2"を選択できる
  - HTTP/3ではトランスポートが異なるため、HTTP/1.1やHTTP/2で接続した後でAlt-Svcを検出することでHTTP/3接続を行う
    - HTTPS RRのalpn値を利用してQUIC接続を確立する場合はこの限りではない
